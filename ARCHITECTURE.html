Narit Vending Machine

System Architecture v2.0

เอกสารสมบูรณ์ อธิบายโครงสร้าง classes, API endpoints, กลไกความปลอดภัย, logging และ flowchart ของระบบควบคุมเครื่องจ่ายสินค้าอัตโนมัติ

🔄 System Flow Diagram

แผนผังแสดงการไหลของข้อมูลและคำสั่งตั้งแต่ผู้ใช้ถึงฮาร์ดแวร์

🖥️ ภาพรวมระบบ

ระบบทำงานบน Raspberry Pi ให้ Pi เป็น web server สำหรับควบคุมเครื่องจ่ายสินค้า ผู้ใช้เปิดหน้าเว็บผ่าน NaritVendingMachine.local หรือเรียกผ่าน REST API โดยตรง

Motion Control
ควบคุมมอเตอร์ X/Y/Z, limit switch, E-Stop, guard ทุก 5 steps, logging
🔗
Command / API
REST API v2.0 (18 endpoints), CORS, input validation, MotionService
📡
MQTT IIoT Layer
Bi-directional Pub/Sub, LWT Heartbeat, Remote Dispense & Status Telemetry
🌐
Web UI
สถานะ realtime poll ทุก 500 ms, ควบคุมผ่าน browser

⚠️ Exception Classes

Class สืบทอดจาก เกิดเมื่อ
MotionError RuntimeError ข้อผิดพลาดทั่วไปของ motion
LimitTriggeredError MotionError ชน limit switch ระหว่างเคลื่อน
EmergencyStopError MotionError E-Stop ถูกกด
NotHomedError MotionError สั่ง move_to_mm โดยยังไม่ home
StopRequestedError MotionError ผู้ใช้กด Stop

📡 การทำงานของระบบ MQTT (MQTT System Operation)

ระบบรองรับการสื่อสารแบบสองทาง (Bi-directional IIoT Protocol) ผ่าน MQTT Broker (เช่น broker.emqx.io) ร่วมกับ REST API เพื่อให้ระบบภายนอก (Payment Gateway, Mobile App, Cloud Dashboard) สั่งจ่ายสินค้าและดึงสถานะตู้ได้แบบ Real-time

Topic Direction QoS Retain คำอธิบาย & Payload ตัวอย่าง
vending/{id}/heartbeat Publish (Pi -> Broker) 1 True สถานะการเชื่อมต่อ (LWT) — {"online": true} หรือ {"online": false} เมื่อเน็ตหลุด/ไฟดับ
vending/{id}/status Publish (Pi -> Broker) 1 True Telemetry พิกัด & สถานะ — {"busy": false, "status": {"state": "idle", "x_mm": 0, ...}}
vending/{id}/response Publish (Pi -> Broker) 1 False ผลลัพธ์ตอบกลับคำสั่ง — {"cmd": "dispense", "result": {"ok": true}}
vending/{id}/cmd/dispense Subscribe (Broker -> Pi) 1 - สั่งจ่ายสินค้าตามช่อง — {"slot": "1"} หรือ {"slot_code": "5"}
vending/{id}/cmd/speed Subscribe (Broker -> Pi) 1 - ปรับความเร็วมอเตอร์ (mm/s) — {"speed_mm_s": 45.0}
vending/{id}/cmd/timer Subscribe (Broker -> Pi) 1 - ตั้งเวลานับถอยหลัง (วินาที) — {"duration_s": 60.0}
vending/{id}/cmd/home Subscribe (Broker -> Pi) 1 - สั่ง Home แกนมอเตอร์ — {"axis": "all"}
vending/{id}/cmd/stop Subscribe (Broker -> Pi) 2 - สั่งหยุดฉุกเฉิน / Soft Stop ทันที — {}
vending/{id}/cmd/clear_alarm Subscribe (Broker -> Pi) 1 - รีเซ็ตสัญญาณแจ้งเตือนและไฟค้าง — {}

🔌 AxisController

ควบคุมมอเตอร์ทีละแกน — pulse GPIO, direction GPIO, อ่าน limit switch, คำนวณ position

เมธอด หน้าที่
move_steps(steps, direction, speed_mm_s) ปล่อย pulse + คำนวณความเร็ว (หน่วงเวลา) อัตโนมัติ
move_mm(distance_mm, speed_mm_s) แปลง mm → steps แล้วเรียก move_steps
move_to_mm(target_mm, speed_mm_s) เคลื่อนไปตำแหน่ง absolute (ต้อง home ก่อน)
home(backoff, max_steps) Home แกน + log start/complete พร้อม step count
stop() ปิด pulse ทันที
status() คืน position, is_homed, head_limit, tail_limit, estop
_guard_before_move() ตรวจ E-Stop, limit, travel range ก่อนเริ่ม
_guard_during_move() ตรวจระหว่างเคลื่อนทุก 5 steps (v2.0: จาก 20)

⚙️ MotionController

รวมการควบคุมแกน X, Y, Z เข้าด้วยกัน จัดการ slot และ stop

เมธอด หน้าที่
axes() คืน {"x":…, "y":…, "z":…}
home_axis(name) Home แกนเดียว
home_all() Home ทุกแกนตาม home_order
move_to(x, y, z, speed_mm_s) เคลื่อน X→Y→Z ตามลำดับ (sequential)
move_by_mm(x, y, z, speed_mm_s) เคลื่อนสัมพัทธ์จากตำแหน่งปัจจุบัน
move_to_slot(code) ยก Z → เคลื่อน X/Y → ลง Z ไปยัง slot
update_slot(code, x, y, z, product_name, dispense_delay_ms) อัปเดตพิกัด + product profile (คง field เดิมถ้าไม่ส่ง)
current_position() คืน {x_mm, y_mm, z_mm} ปัจจุบัน
request_stop() ตั้ง stop flag ทุกแกน
status() คืน estop + X/Y/Z status + current_position

🛡️ กลไกความปลอดภัย

🔴 E-Stop ตรวจก่อนและระหว่างเคลื่อน
🟡 Soft Stop Request จากปุ่ม SOFT STOP (ไม่ block HTTP, มอเตอร์หยุดแต่ไม่ตัดไฟ)
🟠 Limit Switch Min & Max ทุกแกน
🔵 Software Travel Limit (max_travel_mm)
⚡ Guard ทุก 5 steps (จากเดิม 20)
🔒 RLock ป้องกัน 2 motion รันพร้อมกัน
✅ Input Validation ทุก endpoint → 400 แทน 500

🔧 MotionService

Business logic layer ระหว่าง Flask routes และ MotionController — จัดการ lock, error, logging

เมธอด หน้าที่
status_payload() ส่งสถานะทั้งหมด (busy, last_error, slots+product, position)
_run(fn) รัน fn ใน RLock + จัดการ exception + log warning
stop() สั่ง stop ทันที ไม่ใช้ lock — ทำงานได้ขณะ busy
home_axis(axis), home_all() Home แกนเดียว / ทุกแกน
jog(axis, distance_mm) Jog ระยะ relative
move_to(x, y, z) เคลื่อนไปพิกัด absolute
move_to_slot(code) ไปยัง slot
save_slot(code, x, y, z, product_name, dispense_delay_ms) บันทึกพิกัด + product profile → machine_config.json
save_slot_from_current(code) บันทึกตำแหน่งปัจจุบัน → slot
get_slot(code) ดึงข้อมูล slot เดียว + product profile
reset_slot(code) Reset พิกัด → 0,0,0 (คง product_name ไว้)
get_config() ดึง MachineConfig ทั้งหมด
is_axis_homed(axis) เช็กแกนเดียว
is_all_homed() เช็กทุกแกนพร้อมกัน + all_homed flag

🌐 REST API Endpoints (v2.0)

ทุก endpoint คืน "ok": true/false เสมอ — HTTP 200 = สำเร็จ, HTTP 400 = ผิดพลาด

พื้นฐาน

Method Endpoint หน้าที่
GET /api/ping เช็กการเชื่อมต่อ (เร็ว ไม่ query ข้อมูล)
GET /api/status สถานะทั้งหมด (busy, position, slots, estop)
GET /api/config ค่าคอนฟิกแกน X/Y/Z ทั้งหมด

Home

Method Endpoint หน้าที่
POST /api/home/x|y|z Home แกนเดียว
POST /api/home/all Home ทุกแกนตาม home_order
GET /api/home/x|y|z/check เช็ก Home แกนเดียว (ไม่สั่ง Home)
GET /api/home/all/check เช็กทั้ง 3 แกนพร้อมกัน + all_homed

การเคลื่อนที่

Method Endpoint Body หน้าที่
POST /api/jog {"axis":"x", "distance_mm":10, "speed_mm_s":20} Jog แกนเดียว (ระบุ speed_mm_s หรือ time_s ได้)
POST /api/move {"x_mm":10, "y_mm":20, "time_s":5} เคลื่อนไปพิกัด absolute (ระบุ speed_mm_s หรือ time_s ได้)
POST /api/stop — หยุดทันที (ไม่ block)

Slot Management

Method Endpoint Body / หมายเหตุ หน้าที่
GET /api/slots — ดึงข้อมูลทุก slot + product
GET /api/slots/<code> — ดึง slot เดียว
POST /api/slots/<code>/goto — เคลื่อนไปยัง slot
POST /api/slots/<code>/save-current — บันทึกตำแหน่งปัจจุบัน
POST /api/slots/<code> x_mm, y_mm, z_mm, product_name*, dispense_delay_ms* บันทึกพิกัด + product
POST /api/slots/<code>/reset — Reset พิกัด → 0,0,0
DELETE /api/slots/<code> — ลบพิกัด (= reset)

* = optional field

📝 แผนพัฒนาต่อ

Worker Thread — แยก motion เป็น background thread แบบ proper queue สำหรับคำสั่งที่ใช้เวลานาน
Dispense Sequence — ใช้ dispense_delay_ms ที่เก็บไว้ใน slot เพื่อ automate การจ่ายสินค้า
Dispense History — บันทึกประวัติการจ่ายสินค้า (slot, เวลา, สำเร็จ/ล้มเหลว)
Auto-Home on Startup — option --auto-home ใน systemd service
Config Hot-Reload — POST /api/config/reload โดยไม่ต้องรีสตาร์ท service
ทดสอบ steps_per_mm, home_direction, และ max_travel_mm กับเครื่องจริงเสมอ
